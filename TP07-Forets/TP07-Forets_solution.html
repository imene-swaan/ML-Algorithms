II- Forêt aléatoire¶

Voici les tapes de base impliques dans l'excution de l'algorithme de fort alatoire:

  • Choisissez N enregistrements alatoires dans le jeu de donnes.
  • Construisez un arbre de dcision bas sur ces N enregistrements.
  • Choisissez le nombre d'arbres souhait dans votre algorithme et rptez les tapes 1 et 2.
  • Le rsultat :
    • En cas de rgression, pour un nouvel enregistrement, chaque arbre de la fort prdit une valeur pour Y (sortie). La valeur finale peut tre calcule en prenant la moyenne de toutes les valeurs prdites par tous les arbres en fort.
    • En cas de classification, chaque arbre de la fort prdit la catgorie laquelle appartient le nouvel enregistrement. Enfin, le nouvel enregistrement est attribu la catgorie qui remporte le vote la majorit.

def evaluer(nom_modele, modele, X_train, Y_train, X_test, Y_test):

    Y_modele_test = modele.predict(X_test)
    temps_test_2[nom_modele].append(timeit.default_timer() - temps_debut)

    evaluer("foret", foret, X_train, Y_train, X_test, Y_test)

Analyse¶

Dans la phase d'entranement, on remarque que le temps augmente avec le nombre des arbres (puisque le nombre des itrations est gale au nombre des arbres). Aprs un certain nombre d'arbres, on ne peut plus amliorer notre modle (dans notre cas, convergence 100%).

Dans la phase de test, le temps de prdiction augmente aussi. Les cas o on remarque des baisses de temps, ceci puisque l'algorithme apprend des modles adquats au dataset de test (le fait d'entraner un arbre sur un sous ensemble alatoire du dataset d'entranement peut crer de diffrents modles).

La preformance de validation se baisse avec le nombre des arbres (elle augmente des fois). Ceci indique un sur-apprentissage mme si les forts sont destines rgler le probllme de sur-apprentissage chez les arbres de dcision. On peut aussi penser une autre raison qui est le nombre de bruit dans le dataset de test.
