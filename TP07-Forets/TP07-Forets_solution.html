    X_train, X_test, Y_train, Y_test = train_test_split(X,Y,test_size=0.3) # 30% données de test
    
    # ==================================
    # ENTRAINEMENT 
    # ==================================
    
    #entrainement CART
    cart = DecisionTreeClassifier() # un classifieur CART
    temps_debut = timeit.default_timer()
    cart.fit(X_train, Y_train)
    temps_train["cart"].append(timeit.default_timer() - temps_debut)
    
    #entrainement CART7
    cart7 = DecisionTreeClassifier(max_depth=7) # un classifieur CART avec profondeur max de 7
    temps_debut = timeit.default_timer()
    cart7.fit(X_train, Y_train)
    temps_train["cart7"].append(timeit.default_timer() - temps_debut)
    
    
    Y_cart_train = cart.predict(X_train)
    conv["cart"].append(accuracy_score(Y_train, Y_cart_train))
    Y_cart7_train = cart7.predict(X_train)
    conv["cart7"].append(accuracy_score(Y_train, Y_cart7_train))
    
    
    # ==================================
    # TEST 
    # ==================================
    
    #test CART
    temps_debut = timeit.default_timer()
    Y_cart_test = cart.predict(X_test)
    temps_test["cart"].append(timeit.default_timer() - temps_debut)
    
    #test CART7
    temps_debut = timeit.default_timer()
    Y_cart7_test = cart7.predict(X_test)
    temps_test["cart7"].append(timeit.default_timer() - temps_debut)
    
    perf["cart"].append(accuracy_score(Y_test, Y_cart_test))
    perf["cart7"].append(accuracy_score(Y_test, Y_cart7_test))

Ici, nous avons dessiné les métriques par rapport aux runs. L'ordre des runs n'est pas important. Le plus important c'est que un modèle doit être stable par rapport un autre sur les différents runs, sinon il y a une problème quelque part.

Analyse¶

Concernant le temps d'entraînement, on remarque que généralement CART prend plus de temps sans limite de profondeur. Ceci est dû au fait que l'algorithme divise les échantillons jusqu'à arriver à un ensemble homogène ou 2 échantillons par ensemble. Or, dans le cas de la limite de profondeur, on s'arrête même si l'ensemble dans la feuille n'est pas homogène.

Ce qui concerne la converence, CART sans condition a bien pu représenter le dataset d'entraînement, contrairement à celui avec une profondeur délimitée. Ce dernier n'a pas pu converger ce qui indique qu'il existe encore des ensembles non homogènes dans les feuilles.

On veut savoir, aussi, si la meilleure convergence assure une meilleure classification dans le test. Si ce n'ai pas le cas, quel est le problème dans le cas du modèle "cart" ?

Ici, nous avons dessiné les métriques par rapport aux runs. L'ordre des runs n'est pas important. Le plus important c'est qu'un modèle doit être stable par rapport un autre sur les différents runs

Analyse¶

Lorsqu'on fait la prédiction, CART sans condition prend plus de temps. Ceci est logique puisque sa profondeur n'est limitée ce qui veut dire qu'on peut prendre des chemins plus longs afin d'arriver à une décision.

Dans la plupart du temps, CART sans condition mène à une précision plus grande que celle du CART avec une profondeur fixe. Dans ce cas, on peut conclure que ce premier a pu généraliser mieux que le deuxième. Mais, aussi, on remarque qu'il y a des cas où CART non conditionné peut donner moins de performance lors du test. Ceci indique qu'il y a peut-être un problème de sur-apprentissage : l'algorithme essaye de représenter le dataset de l'entraînement en découpant les groupes à chaque fois, ce qui mène à un modèle moins général (qui marche mieux avec le dataset d'entraînement ou un dataset proche)

II- Forêt aléatoire¶

Voici les étapes de base impliquées dans l'exécution de l'algorithme de forêt aléatoire:

  • Choisissez N enregistrements aléatoires dans le jeu de données.
  • Construisez un arbre de décision basé sur ces N enregistrements.
  • Choisissez le nombre d'arbres souhaité dans votre algorithme et répétez les étapes 1 et 2.
  • Le résultat :
    • En cas de régression, pour un nouvel enregistrement, chaque arbre de la forêt prédit une valeur pour Y (sortie). La valeur finale peut être calculée en prenant la moyenne de toutes les valeurs prédites par tous les arbres en forêt.
    • En cas de classification, chaque arbre de la forêt prédit la catégorie à laquelle appartient le nouvel enregistrement. Enfin, le nouvel enregistrement est attribué à la catégorie qui remporte le vote à la majorité.

# cette fonction utilise les variables globales ci-avant
def evaluer(nom_modele, modele, X_train, Y_train, X_test, Y_test):
    # temps entrainement
    temps_debut = timeit.default_timer()
    modele.fit(X_train, Y_train)
    temps_train_2[nom_modele].append(timeit.default_timer() - temps_debut)
    
    # converence
    Y_modele_train = modele.predict(X_train)
    conv_2[nom_modele].append(accuracy_score(Y_train, Y_modele_train))
    
    # temps de test
    temps_debut = timeit.default_timer()
    Y_modele_test = modele.predict(X_test)
    temps_test_2[nom_modele].append(timeit.default_timer() - temps_debut)
    
    # performance
    perf_2[nom_modele].append(accuracy_score(Y_test, Y_modele_test))
    

for run in range(NBR_RUN_2): 
    # prétaitement des données
    X_train, X_test, Y_train, Y_test = train_test_split(X,Y,test_size=0.3) # 30% données de test
    
    cart_2 = DecisionTreeClassifier() # un classifieur CART
    evaluer("cart_2", cart_2, X_train, Y_train, X_test, Y_test)
    
    cart7_2 = DecisionTreeClassifier(max_depth=7) # un classifieur CART avec profondeur max de 7
    evaluer("cart7_2", cart7_2, X_train, Y_train, X_test, Y_test)
    
    nb = GaussianNB() # un classifieur Naive Bayes pour les caractéristiques numériques
    evaluer("nbayes", nb, X_train, Y_train, X_test, Y_test)
    
    rl = LogisticRegression(multi_class="multinomial", solver="lbfgs", max_iter=1000) # un classifieur de régression logistique
    evaluer("reglog", rl, X_train, Y_train, X_test, Y_test)
    
    foret = RandomForestClassifier(n_estimators=100, random_state=0) # un classifieur de foret avec 100 arbres
    evaluer("foret", foret, X_train, Y_train, X_test, Y_test)
    
print("FIN EVALUATION")

Analyse¶

On commence par analyser le temps d'entraînement. On remarque que Naive Bayes et CART ont des temps d'entraînement comparables, les forêts sont plus lents et la régression logistique est la plus lourde. Les forêts prennennt plus de temps dans l'entraînement puisqu'on est entrain d'entraîner plusieurs arbres de décision d'une façon séquencielle. Une proposition pour améliorer leur temps d'entraînement est d'utiliser le parallélisme. La régression logistique prend plus de temps puisqu'elle utilise des calculs plus complexes et sa complexité augmente avec le nombre des caractéristiques en plus le nombre des itérations.

En comparant les forêts avec les autres algorithmes dans ce dataset, on constate que les forêts convergent mieux vers la solution optimale avec moins de temps.

II-5- Analyse de l'effet du nombre des arbres dans un forêt¶

Ici, le nombre des arbres dans le forêt est varié de 10 à 100 avec un pas de 5.

Analyser les résultats ci-après

  • Temps d'entraînement par rapport au nombre des arbres
  • La convergence par rapport au nombre des arbres
  • Le temps de prédiction ar rapport au nombre des arbres
  • La performance par rapport au nombre des arbres

Analyse¶

Dans la phase d'entraînement, on remarque que le temps augmente avec le nombre des arbres (puisque le nombre des itérations est égale au nombre des arbres). Après un certain nombre d'arbres, on ne peut plus améliorer notre modèle (dans notre cas, convergence 100%).

Dans la phase de test, le temps de prédiction augmente aussi. Les cas où on remarque des baisses de temps, ceci puisque l'algorithme apprend des modèles adéquats au dataset de test (le fait d'entraîner un arbre sur un sous ensemble aléatoire du dataset d'entraînement peut créer de différents modèles).

La preformance de validation se baisse avec le nombre des arbres (elle augmente des fois). Ceci indique un sur-apprentissage même si les forêts sont destinées à régler le probllème de sur-apprentissage chez les arbres de décision. On peut aussi penser à une autre raison qui est le nombre de bruit dans le dataset de test.

Le temps d'entraînement et de test sont variables ; il ne suivent pas la profondeur maximale (elle peut avoir un effet minimal par rapport au partionement aléatoire du dataset).

Concernant la précision du modèle (accuracy), on remarque que le modèle converge mieux lors de l'entraînement lorsqu'on augmente la profondeur maximale de ces arbres. Dans la phase de test, c'est la même remarque. Dans notre cas, augmenter la profondeur maximale n'entraine pas un sur-apprentissage dans le modèle.
